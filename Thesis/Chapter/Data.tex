\subsection{Introduction to the Datasets}
\label{data_description}

We will analyse labelling uncertainty in the context of image classification
using the \textit{CIFAR-10H} dataset. The dataset is based on the
\textit{CIFAR-10} (Canadian Institute for Advanced Research, 10 classes)
dataset, first introduced in \cite{Krizhevsky2009LearningML}, which is a
benchmark dataset for image classification and is itself a subset of the larger
\textit{80 million tiny images} dataset, see \cite{torralba80MillionTiny2008}.
The \textit{CIFAR-10} dataset consists of 60'000 images of size $32 \times 32$
pixels. Each image belongs to one of ten classes: airplane, automobile, bird,
cat, deer, dog, frog, horse, ship, and truck. Some example images of each class
are shown in Figure \ref{fig:cifar10example}. The images in the
\textit{CIFAR-10} dataset come with multiple advantages for our purposes, see
\cite{petersonHumanUncertaintyMakes2019}. First, the dataset is well-known and
widely used in the machine learning community, see \cite{PapersCodeCIFAR10} for
a list of papers that use the dataset. Second, the low resolution of the images
makes the dataset challenging for image classification and introduces ambiguity
in the labelling process. Third, the dataset contains images that are close to
the category boundaries, which further introduces ambiguity in the labelling
process. This is in contrast with other datasets that are more carefully created
such that each image is selected to be a unabiguously of a certain
category.\bigskip

The \textit{CIFAR-10} dataset is divided into a training set of 50'000 images
and a test set of 10'000 images. The test set is the basis for the
\textit{CIFAR-10H} dataset, which will be used in this thesis.
\textit{CIFAR-10H} is a balanced dataset, meaning that each class contains the
same number of images, i.e. 1000 images. \textit{CIFAR-10H} is a human-labelled
dataset, where each image has been labelled by multiple human annotators. The
dataset contains 514'200 observations from a total of 2571 annotators. Each
annotator labelled 200 images, 20 from each category, which were randomly
sampled from the test set of the \textit{CIFAR-10} dataset. On average, each
image was labelled 51 times, with the minimum number of labels being 50 and the
maximum number of labels being 63. The relevant features of the
\textit{CIFAR-10H} dataset are described in Table \ref{tab:cifar10hvariables}.
The study design is described in more detail in
\cite{petersonHumanUncertaintyMakes2019}, section 4.2. Note, that the images
where collected based on searching the web, using search engines. Thus, the true
label is not necessarily the object that is displayed in the image, but the term
that was used to search for the image, see \cite{torralba80MillionTiny2008}. For
our purposes, we will assume that the true label is the object that is displayed
in the image.
\bigskip

The \textit{CIFAR-10} and \textit{CIFAR-10H} datasets are publicly available and
can be downloaded from the CIFAR-10 website \cite{CIFAR10CIFAR100Datasetsb} and the
author's GitHub repository \cite{JcpetersonCifar10hOriginal}.

\subsection{Descriptive Statistics}
\subsubsection{Label Ambiguity}
\label{desc_stats}

As mentioned in the previous section, the \textit{CIFAR-10H} dataset is balanced
with respect to the true categories. This means, that we have 51'420
observations per true category. But the number of observations per chosen
category is not balanced, see Figure \ref{fig:chosen_category}, e.g. the
category \textit{deer} is underrepresented, while the category \textit{dog} is
overrepresented. The question that arrises is, whether chosen categories are
usually correct or not and where the votes of overrepresented categories come
from.

\begin{figure}[H]
  \centering
  \includegraphics[width = 0.8\textwidth]{../figs/chosen_category.png}
  \caption{Number of observations per chosen category. The blue horizontal line
  indicates the 51'420 observations per true category, so that we can see, which
  categories are over- and underrepresented.}
  \label{fig:chosen_category}
\end{figure}

We can also question the ambiguity of the images. For that, we can check, how
many unique labels each image has received. If an image is unambiguous, then all
annotators should agree on the label of the image. If an image is ambiguous,
then there are multiple possible labels for the image. In Figure
\ref{fig:consensus_distribution}, we can see the distribution of the number of
labels per image for each true category. We can see, that in all categories,
50\% of the images have received at most 2 labels. We can also see, that in
almost all categories, 25\% of the images have received 3 labels or more. This
indicates that a lot of images are unambiguous, but also that a considerable amount of
images are ambiguous. In Figure \ref{fig:ambiguous_image}, we can see an examples of ambiguous and
unambiguous images as well as their respective vote distribution.\bigskip

Another possibility to check the ambiguity of the images is to check the
entropy of the vote distribution. The entropy of an image is defined as
\begin{equation}
  H = - \sum_{i = 1}^{10} p_{i} \log_2(p_{i}),
  \text{ with } p_{i} = \frac{n_{i}}{N},
  \label{eq:entropy}
\end{equation}
and $n_{i}$ is the number of votes for category $i$ and $N$ is the total number
of votes that the image received. A high entropy indicates that the image is
ambiguous, while a low entropy, i.e. close to 0, indicates that the image is
unambiguous. In Figure \ref{fig:entropy_distribution}, that is replicated from
\cite{battledayCapturingHumanCategorization2020a}, we can see the distribution
of the entropy and can again confirm that most images are rather unambiguous,
but that a considerable amount of images are ambiguous. Note, that the entropy
can be at most $3.33$, because that's the value under the assumption that
the votes for a certain image are uniformly distributed.

\begin{figure}[ht]
  \centering
  \includegraphics[width = 0.8\textwidth]{../figs/consensus_distribution.png}
  \caption{Distribution of the number of unique chosen labels per image for each true category.
  Unambiguous images have received only one label, while ambiguous images have received multiple labels.
  The more labels an image has received, the more ambiguous the image is.
  }
  \label{fig:consensus_distribution}
\end{figure}

This ambiguity in the labelling process is also reflected in the confusion
matrix in Figure \ref{fig:confusion_matrix}. The confusion matrix shows the
propotion of votes for each chosen category given the true category. We can see
that the diagonal of the confusion matrix is not 1, because the annotators do
mislabel the image. We can also see that the confusion matrix is not symmetric,
which indicates that the mislabelling is not symmetric, i.e. the annotators do
not confuse the categories in both directions equally. For example, a deer is
more often confused with a horse than a horse is confused with a deer. We can
also see, that the confusion matrix is almost sparse, i.e. there are a lot of
values that are almost 0, which indicates that the annotators do not confuse the
categories randomly, but that there are some categories that are more often
confused with each other. One goal of this thesis is to estimate the confusion
matrix from Figure \ref{fig:confusion_matrix} by only using the chosen
categories.\bigskip

Lastly, we can check the number of subcategories per true category. The
subcategory is a more detailed label of the image. The subcategories are not
balanced, see Table \ref{tab:subcategory}. On average, there are
$16$ images per subcategory, but there is a lot of variance in the number of
images per subcategory. The distribution of the number of images per subcategory
is shown in Appendix \ref{app}, Figure \ref{fig:subcategory_distinct_image_dist}.
One question of interest is, whether there are subcategories which more
ambiguous images, i.e. with more labelling uncertainty, than others.

\subsubsection{Reaction Time and Elapsed Time}

Another interesting feature of the \textit{CIFAR-10H} dataset is the reaction
time of the annotators. The reaction time is the time that the annotator needed
to provide a response for the current trial. In Figure \ref{fig:reaction_time},
we can see the distribution of the reaction time (in seconds). The median
reaction time is around 1.48 seconds.

\begin{figure}[H]
  \centering
  \includegraphics[width = 0.8\textwidth]{../figs/Reaction_Time_Dist.png}
  \caption{Distribution of the reaction time (in seconds) for each true category. 
  The median reaction time is around 1.48 seconds. We cut off the
  observations that have a reaction time of more than 6 seconds, which was approx.
  1.9\% of the observations.}
  \label{fig:reaction_time}
\end{figure}

One interesting aspect of the reaction time is the relationship between the
reaction time and the entropy of the vote distribution. Computing the
correlation between the reaction time and the entropy of the vote distribution
on the whole dataset, we get a correlation of $0.0372$. But this correlation is
misleading, because we have a lot of outliers in the reaction time, which are
not necessarily indicative of labelling uncertainty, e.g. the longest observed
reaction time is around 2.13 hours with an image entropy of 0. We therefore need
to remove the outliers from the dataset, but it is not clear, how to define an
outlier in this context. In Figure \ref{fig:reaction_time_entropy}, we can see
the pearson-correlation between the reaction time and the entropy of the vote
distribution based on the cutoff-value of the reaction time, beginning at 6
seconds. We can see, that the correlation is increasing first, up to a value of
$0.32$ at $11$ seconds, and then it decreases. Using a threshold of $26$ seconds
for the reaction time for later analysis seems reasonable, because this is above
the 99.9\% quantile of the reaction time distribution. This results in a correlation
of $0.30$ and it is reasonable to assume that that's enough time to label even
the most ambiguous images.

\begin{figure}[H]
  \centering
  \includegraphics[width = 0.8\textwidth]{../figs/entropy_reaction_time_cor.png}
  \caption{Pearson-correlation between the reaction time and the entropy of the 
  vote distribution based on the cutoff-value of the reaction time.}
  \label{fig:reaction_time_entropy}
\end{figure}

Another interesting aspect of the reaction time is the relationship between the
reaction time and the correctness of the chosen category. In Figure
\ref{fig:Reaction_Time_Correct_Guess}, we can the relationship between the
reaction time and the correctness of the chosen category. We can see that for a
higher reaction time, the probability of a correct guess is becomes lower. This
will be further investigated in the next sections. For now, this is just a
indicator, that a higher reaction time might be an indicator for labelling
uncertainty. The question that arises is, whether the decreased probability of a
correct guess is due to the fact, that some images are more ambiguous than
others, thus more prone to labelling uncertainty, or whether there are
annotators that are more prone to labelling uncertainty than others.\bigskip

One issue that may arise in regards to the quality of the data are annotators
that are not paying attention and are just randomly guessing the labels. This is
supposed to be controlled for in the study design, by performing so called
\textit{attetion checks}, where the annotators receive an image with a clear
unambiguous label and are asked to label the image with the correct label. If
the annotator failed to do so, the annotator was excluded from the study.
Nonetheless, by random chance, there could still be some annotators that guessed
the labels correctly and stayed in the study. In Figure
\ref{fig:elapsed_time_vs_correct}, we can see the total elapsed time, i.e. the
time that the annotator needed to label all 200 images, for each annotator(in
minutes) vs. the proportion of correct guesses. We can see that the proportion
of correct guesses is almost always above 80\% and that the elapsed time is
almost always between 5 and 20 minutes. We can also see that the annotators with
a lower proportion of correct guesses tend to need have a lower total elapsed
time. Especially the annotators with a proportion of correct guesses below 25\%
were very fast in labelling the images, with an average total elapsed time of
around 4.6 minutes, which is less than 1.4 seconds per image for all 200 images.
